**Contributing GuidelinesContributing Guidelines**
Welcome
Thank you for your interest in KernelCTC! This is a solo-maintained project, but contributions are welcome — whether it's a bug report, a suggestion, or a pull request.
How to Contribute
Report a bug

Open a GitHub issue with a clear description of the problem, your Python version, and OS. Screenshots help.
Suggest a feature

Open an issue with the label enhancement. Explain what it does and why it would be useful.
Submit a pull request

Fork the repo and create a new branch: git checkout -b feature/your-feature
Keep changes focused — one feature or fix per PR
Test your changes before submitting
Write a clear PR description explaining what changed and why

Code Style

Follow PEP 8
Keep everything in a single file (kernelctc.py) unless there's a strong reason to split
Add comments for any non-obvious tax logic

What I'll Accept

Bug fixes
UI improvements
Tax calculation accuracy fixes
Support for additional deductions or regimes

What I Won't Accept

Breaking changes without discussion
Unrelated refactors in the same PR
Dependencies that require installation (project stays zero-install)
